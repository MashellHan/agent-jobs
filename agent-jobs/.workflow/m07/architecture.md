#### 1.1.1 Custom-symbol vs. PNG decision (architect's call)

Spec competitive-analysis §4 lays out two paths for the menubar glyph: (a) custom
SF Symbol `.symbolset` or (b) raw PNG image set with `isTemplate = true`.

**Architect picks (b) raw PNG image set.** Rationale:

1. SwiftPM's resource pipeline supports `.imageset` directly today; `.symbolset`
   support is partial in SPM 6.0 + macOS 14 (no `.car` symbol indexing in the
   non-Xcode build). Going (a) risks a 30%-of-the-time render failure depending
   on toolchain.
2. Spec's contract is a 16pt template image, not a Dynamic-Type-scaled symbol.
   We don't gain much from SF Symbols' weight × scale grid for a fixed 16pt
   menubar slot.
3. PNG gives deterministic per-rung control — a designer hand-tuning the @1x
   versus @2x PNG hairlines is strictly better at 16pt than a single SVG getting
   downscaled by SF Symbols rendering.

Cost: we lose Dynamic-Type auto-scaling. Acceptable per spec (out-of-scope:
density / accessibility deferred to M14).

## 2. T-001 menubar icon — design contract & badge composition

### 2.1 Glyph design contract (for whoever exports the SVG)

- 16pt logical, exported at @1x (16×16), @2x (32×32), @3x (48×48).
- Single-layer black on clear; AppKit's template-render path strips color and
  tints by appearance (see Stats lesson, competitive-analysis §1).
- Monoline / monoshape; avoid hairlines < 1pt @1x — they vanish under dark-menubar tint.
- Metaphor: "watching eye + task list" or similar (per spec §"Identity strength"
  reference to Bartender's B-with-bar). Final glyph design is a designer call;
  architecture does not specify pixels.
- **Done-when (test-verifiable)**: AC-F-04 confirms the 3 PNGs + manifest exist;
  AC-F-06 confirms the rendered image is template; AC-V-01/04 confirm luma
  inverts correctly between light/dark menubar.

### 2.2 Badge composition (running-count overlay)

Spec contract:
- `runningCount == 0` → no badge layer rendered. Idle glyph only.
- `1 ≤ runningCount ≤ 9` → literal digit overlay in trailing slot.
- `runningCount ≥ 10` → `"9+"` literal in trailing slot.

Implementation in `MenuBarLabel.swift`:

```swift
struct MenuBarLabel: View {
    let state: MenuBarSummary
    var body: some View {
        HStack(spacing: DesignTokens.Spacing.xs) {
            Image("MenuBarIcon", bundle: .main)
                .renderingMode(.template)   // AC-F-06
            BadgeOverlay(count: state.running)
                .accessibilityHidden(state.running == 0)
            if state.failed > 0 {
                Image(systemName: "exclamationmark.triangle.fill")
                    .foregroundStyle(DesignTokens.SemanticColor.statusFailed)
                    .imageScale(.small)
            }
        }
        .accessibilityIdentifier("agent-jobs.menubar")
        .accessibilityLabel("Agent Jobs menu bar status — \(state.running) running")
    }
}

private struct BadgeOverlay: View {
    let count: Int
    var body: some View {
        if count == 0 { EmptyView() }
        else {
            Text(count >= 10 ? "9+" : "\(count)")
                .font(.system(.caption2, design: .rounded, weight: .semibold))
                .monospacedDigit()
                .foregroundStyle(.primary) // NOT a template; takes appearance
                .accessibilityIdentifier("agent-jobs.menubar.badge")
        }
    }
}
```

**No bitmap badge image** — text composition per iStat Menus lesson
(competitive-analysis §3). Three unit tests one per branch satisfy AC-F-07.

### 2.3 .icns build script

`scripts/build-icns.sh` is a single bash file. Pseudocode:

```bash
#!/bin/bash
set -euo pipefail
SRC_APP_SVG="Sources/AgentJobsMacApp/Resources/Identity/app-icon.svg"
SRC_MENU_SVG="Sources/AgentJobsMacApp/Resources/Identity/menubar-glyph.svg"
APPICON_OUT="Sources/AgentJobsMacApp/Resources/Assets.xcassets/AppIcon.appiconset"
MENUICON_OUT="Sources/AgentJobsMacApp/Resources/Assets.xcassets/MenuBarIcon.imageset"
ICNS_OUT=".build/agent-jobs.icns"
# Render app-icon at 10 sizes (16, 32, 128, 256, 512 @1x/@2x):
for size in 16 32 128 256 512; do
  rsvg-convert -w $size -h $size  "$SRC_APP_SVG" > "$APPICON_OUT/icon_${size}.png"
  rsvg-convert -w $((size*2)) -h $((size*2)) "$SRC_APP_SVG" > "$APPICON_OUT/icon_${size}@2x.png"
done
# Render menubar glyph at 3 sizes:
rsvg-convert -w 16 -h 16 "$SRC_MENU_SVG" > "$MENUICON_OUT/menubar-glyph.png"
rsvg-convert -w 32 -h 32 "$SRC_MENU_SVG" > "$MENUICON_OUT/menubar-glyph@2x.png"
rsvg-convert -w 48 -h 48 "$SRC_MENU_SVG" > "$MENUICON_OUT/menubar-glyph@3x.png"
# Build .icns from a temp iconset (per Apple iconutil):
TMPSET=$(mktemp -d)/agent-jobs.iconset
mkdir -p "$TMPSET"
cp "$APPICON_OUT/icon_"*.png "$TMPSET/"
iconutil --convert icns "$TMPSET" --output "$ICNS_OUT"
echo "wrote $ICNS_OUT"
```

`rsvg-convert` is on most dev machines via `brew install librsvg`. Fallback:
swap in `qlmanage -t -s $size -o tmp/ "$SRC_APP_SVG"` (Quick Look thumbnailer,
no brew). Implementer documents the brew dep in `Resources/Identity/README.md`.

**Idempotency**: `rsvg-convert` is deterministic; running the script twice
produces byte-identical PNGs as long as the SVG is unchanged. AC-F-14 (capture-all
byte-stable rerun) extends transitively — if the asset PNGs don't churn, neither
does the rendered menubar scenario.

### 3.2 New `HarnessScenes` factories

Added to `Sources/AgentJobsMacUI/AgentJobsMacUI.swift` `HarnessScenes` enum:

```swift
public extension HarnessScenes {
    enum IconState { case idle, running(Int) }

    /// Renders MenuBarLabel at a fixed 22×22 frame matching the macOS
    /// status-item bounding box. Used for AC-V-01..04.
    static func menuBarIconOnly(state: IconState) -> AnyView {
        let summary: MenuBarSummary
        switch state {
        case .idle:               summary = .empty
        case .running(let n):     summary = MenuBarSummary(running: n, …)
        }
        return AnyView(
            MenuBarLabel(state: summary)
                .frame(width: 22, height: 22)
                .padding(0)
        )
    }

    /// Renders the full token swatch composite — color chips + type-scale
    /// specimen + spacing-scale ruler. Used for AC-V- (no diff AC, only
    /// AC-D-03 rubric) at scenario 14.
    static func tokensSwatch(size: CGSize = CGSize(width: 800, height: 600)) -> AnyView {
        AnyView(TokensSwatchView().frame(width: size.width, height: size.height))
    }
}
```
