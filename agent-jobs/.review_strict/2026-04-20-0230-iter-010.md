## M-005 Implementation detail

**`scripts/pre-commit-gate.sh`** (52 LOC):
- Reads staged file list via `git diff --cached --name-only`
- If no `macapp/AgentJobsMac/` files staged → **skip** (avoids 5s cost on README/docs commits)
- Otherwise: `swift build` then `swift test`; non-zero exit → block commit
- Output prefixed with `[pre-commit-gate]` for clear tooling provenance

**`.claude/settings.json`** addition:
```json
"PreToolUse": [{
  "matcher": "Bash(git commit:*)",
  "hooks": [{
    "type": "command",
    "command": ".../scripts/pre-commit-gate.sh",
    "timeout": 120000
  }]
}]
```

Existing PostToolUse `auto-commit.sh` hook left intact.

**Why this resolves M-005:**
- iter-005 root cause: implementer claimed "build passing" while a stale build cache hid a red `swift build`. Then committed.
- With this hook, **no `git commit` for macapp changes can succeed unless a fresh `swift build && swift test` passes in this turn**. The cache cannot lie because the gate runs the toolchain itself.

**Limitations / known gaps:**
1. The hook only fires when Claude Code invokes `git commit` via Bash. A human running `git commit` from a separate shell bypasses it. → acceptable for an agent-driven loop; to harden, a real `.git/hooks/pre-commit` symlink would be needed.
2. 120s timeout — should be enough for ~2s build + 0.3s test, but if the project grows (e.g. integration tests added) this may need bumping.
3. Doesn't gate `--amend` or `commit -F` — matcher is broad enough (`git commit:*`) to catch most agent-driven invocations though.
