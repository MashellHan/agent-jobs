## Code Review — 新代码

### `Shell.swift` (119 行) ⭐

履行 sandbox-decision.md 第 38-39 行承诺，质量出众：

- ✅ `precondition(executable.hasPrefix("/"))` — 拒绝 PATH lookup，强制绝对路径，可审计
- ✅ argv array 显式参数，文档明说"whitespace in entry preserved verbatim, no shell parsing"
- ✅ `withThrowingTaskGroup` race process 与 timer，winner cancel 兄弟
- ✅ `withTaskCancellationHandler` 双阶段 reap：SIGTERM → 0.5s grace → SIGKILL
- ✅ stdout/stderr 分别 `Pipe()` 并在 `terminationHandler` 内一次性 read（避免 deadlock）
- ✅ `Failure` enum 类型化错误：`.timeout(seconds:)` / `.launchFailed(String)` / `.nonZeroExit(code:stderr:)`
- ✅ `defaultTimeoutSeconds = 5` 静态常量，长任务必须显式 opt-in
- ⚠️ **小观察**：`runProcess` 内的 `try? readToEnd()` 失败静默 — 若 stdout 是大文件 OOM 会失数据。discovery 命令产物小，可接受。
- ⚠️ **小观察**：`onCancel` 用 `DispatchQueue.global().asyncAfter` 跳出 structured concurrency 做 SIGKILL；可以但混用。换 `Task.detached { try? await Task.sleep; ... kill(...) }` 更纯 Swift Concurrency。**LOW**

### `AgentJobsJsonProvider.readWithTimeout` 

- ✅ Pattern 与 `Shell.run` 一致（TaskGroup race），代码统一
- ✅ Child task `priority: .utility` 不抢占 UI
- ✅ `ProviderError.timeout` 显式路径 + logger.error
- ⚠️ 仍是 `Data(contentsOf:)` 同步 IO，只是被 child task 包裹 — timeout 触发后 sync read 还在跑直到完成（线程占用）。对 jobs.json 这种 KB 级文件可接受；若未来读 MB 级 plist 应改 `FileHandle.AsyncBytes`。**LOW**

### `ServiceRegistryTests.swift` (95 行) ⭐

- ✅ 3 个 stub provider（Good/Failing/Slow）覆盖 happy / 失败隔离 / 并行
- ✅ "failing provider does NOT poison the rest" 测试用 Slow + Failing + Good 组合，**真正验证 TaskGroup 并行 + 隔离**而非 sequential
