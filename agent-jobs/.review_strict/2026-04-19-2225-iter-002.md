## Build & Test

```
$ swift build
warning: 'agentjobsmac': Invalid Resource 'Fixtures': File not found.
Build complete! (0.15s)   # incremental — fresh clean build later in same run reported success
$ swift test
error: no such module 'XCTest'   # initial run
error: no such module 'Testing'  # 当前 Package.swift 已去掉 resources，改了 testTarget 配置
```

| Check | Result |
|-------|--------|
| `swift build` | ⚠️ 通过但有警告（Fixtures 资源声明指向不存在的目录） |
| `swift test` | 🚫 **FAIL — 0 测试可跑** |

### Build warning 根因

`Package.swift` 原来写：
```swift
.testTarget(... resources: [.copy("Fixtures")])
```
但 `Tests/AgentJobsCoreTests/Fixtures/` 不存在。现 Package.swift 已被修正去掉 `resources`，但 build cache 仍 surface 警告。

### Test failure 根因

测试源码用 `import XCTest`，而 Package.swift / 当前工程上下文期望 `swift-testing` 框架（`import Testing`）。两套测试框架混用，testTarget 没有 explicit linkage，导致两种 import 都报 "no such module"。

**这是 release-blocking。** 一个连测试都跑不起来的 commit 在主干上是 CRITICAL 级别问题。
