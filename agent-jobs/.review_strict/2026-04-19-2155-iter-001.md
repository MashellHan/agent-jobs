## Baseline expectations from existing TUI

源 TUI 项目（参考 `src/` 共 ~4934 行 TS + Ink）已确立的契约必须在 Mac app 中保持：

### 数据模型（来自 `src/types.ts`、README）

4 种 source，每条 service record 至少包含：

- `id`, `name`, `source` ∈ {registered, live, cron, launchd}
- `status`, `pid?`, `port?`
- `command`, `cwd?`
- `created_at` (epoch / ISO) — **必须显示**（MEMORY: feedback_tui_history）
- `schedule?` — cron 表达式或 launchd interval（按真实频率显示，MEMORY: feedback_schedule_display）
- `history[]` — 友好截断（MEMORY: feedback_tui_history）
- `origin?` — agent / tool / session 来源（registered 专属）

### 4 数据源后端调用

| Source | 调用 | 风险点 |
|--------|------|--------|
| registered | 读 `~/.claude/...` JSON 文件 | 文件锁、JSON 损坏、原子写 |
| live | `lsof -i -P -n -sTCP:LISTEN` | 沙盒（macOS app sandbox 不允许）、parsing |
| cron | 读 `~/.claude/scheduled_tasks.json` | 同 registered |
| launchd | 扫 `~/Library/LaunchAgents/*.plist` + `plutil` | 沙盒、用户/系统域 |

⚠️ **macOS App Sandbox 关键约束**：若 app 走 Mac App Store / 默认沙盒，`lsof`、`launchctl`、读他人目录都会被拒。必须明确决策：
- **Option A**：Developer ID 分发，关闭沙盒（开发者工具常见做法）
- **Option B**：保留沙盒，仅访问 `~/Library/Application Support/`，把 `lsof` 等放到 helper tool / XPC service
- **Option C**：保留沙盒，加 `com.apple.security.temporary-exception.files.home-relative-path.read-only` 等临时豁免

下一轮必须看到这个决策。
