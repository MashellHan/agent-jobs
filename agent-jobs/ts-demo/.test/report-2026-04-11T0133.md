## Issues Found & Fixed

### 1. Test isolation bug in detect.test.ts (FIXED)

**Root cause:** The "deduplicates by name" test overrides `existsSync` and `readFileSync` mock implementations via `mockReturnValue`. The subsequent "extracts port" test uses `clearAllMocks` which clears call history but NOT mock implementations. This caused `loadJobs()` to return stale job data from the dedup test, making `flask-server` appear as `jobs[1]` instead of `jobs[0]`.

**Fix:** Changed the assertion from `written.jobs[0].port` to `written.jobs.find(j => j.name === "flask-server").port` to be order-independent.
