### 3.3 Pattern matching edge cases (detect.ts)

**Potential issues found:**

1. **detect.ts:47** -- `docker run -d` regex requires a space after `-d`:
   ```
   re: /docker\s+run\s+(?:.*\s)?-d\s/i
   ```
   The trailing `\s` means `docker run -d` at the end of a command (no trailing space) would NOT match. However, a bare `docker run -d` without an image name is invalid Docker usage, so this is acceptable behavior. If the image is present (`docker run -d nginx`), there is always a space after `-d`.

2. **detect.ts:52** -- Docker image fallback regex may not capture correctly:
   ```
   cmd.match(/docker\s+run\s+(?:.*\s)(\S+)\s*$/)
   ```
   The non-greedy middle part `(?:.*\s)` tries to skip flags to find the last non-flag argument. For a command like `docker run -d -e VAR=1 nginx`, this captures `nginx` correctly. But for `docker run -d nginx:latest arg1 arg2`, it would capture `arg2` instead of `nginx:latest`. Edge case, unlikely in real usage since Docker images are typically the last positional argument.

3. **detect.ts:95-101** -- `node` and `python` patterns only match specific extensions:
   ```
   re: /\bnode\s+(\S+\.(?:js|mjs|ts))\b/i
   re: /\b(python3?)\s+(\S+\.py)\b/i
   ```
   This means `node server.cjs` would NOT match. CJS is rare in new projects, but worth noting.

4. **detect.ts:222** -- The `run_in_background` check serializes the entire input:
   ```typescript
   const isBackground = BACKGROUND_RE.test(cmd) || /run_in_background/.test(JSON.stringify(input));
   ```
   This works but could match false positives if `"run_in_background"` appears as a substring in the command text or output. Low risk in practice.

**loader.ts:21-38** (`loadRegisteredJobs`):

### 3.5 Logic bugs

**No logic bugs found.** The code is well-structured and the test suite covers the critical paths thoroughly.

One **potential behavioral issue**: `detect.ts:178` deduplicates by `name` only. Two different services with the same name in different projects would be treated as duplicates. For example:
- Project A: `pm2 start api.js` -> name `pm2 api.js`
- Project B: `pm2 start api.js` -> name `pm2 api.js` -- would be silently dropped

Review-010 has this as M3 (MEDIUM, dedup uses name only).

### 3.6 Other observations

**utils.ts:89-92** (`truncate`):
- `truncate("abc", 0)` returns `"…"` (length 1), which exceeds the requested max of 0. Not a practical issue since truncate is never called with max=0 in the codebase, but mathematically incorrect.

**job-detail.tsx:27-32** -- Port and PID are shown conditionally:
```typescript
if (job.port) { ... }
if (job.pid) { ... }
```
Port `0` and PID `0` would be treated as falsy and not shown. Port 0 is meaningless, but PID 0 (the kernel scheduler on Unix) being hidden is technically a bug -- though no real service would have PID 0.

**app.tsx:105-113** -- Cursor movement does not wrap at boundaries (no cycling from last to first). This is an intentional UX choice, not a bug.
