3. **setup.ts has no test file** -- The loadSettings() try/catch added in v013 review has zero test coverage. This is a correctness gap: corrupt `settings.json` handling is only validated by manual inspection.

5. **`readFileSync(0)` for stdin in main()** -- Works but is non-standard. Documented as H4.

6. **Dedup key uses `name` only** -- Two different services with the same name from different projects would collide. Documented as M3.
